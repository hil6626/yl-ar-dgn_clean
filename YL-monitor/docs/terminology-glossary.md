## 一、核心概念术语

| 中文术语 | 英文对照 | 定义 | 使用场景 |
|----------|----------|------|----------|
| 监控 | Monitor | 对系统、服务、资源进行持续观察和记录 | 系统监控、资源监控 |
| 仪表盘 | Dashboard | 可视化展示监控数据和状态的界面 | 前端页面、数据展示 |
| 告警 | Alert | 当指标超过阈值时发出的通知 | 告警系统、通知机制 |
| 脚本 | Script | 可执行的自动化任务程序 | 脚本管理、任务执行 |
| DAG | DAG (Directed Acyclic Graph) | 有向无环图，用于表示任务依赖关系 | 工作流管理、任务编排 |
| 节点 | Node | DAG中的单个任务单元 | DAG可视化、任务管理 |
| 边 | Edge | DAG中连接节点的依赖关系 | DAG构建、依赖定义 |
| 沉积内容 | Stale Content | 过期、无用或不再需要的文件和数据 | 清理管理、存储优化 |
| 重复内容 | Duplicate Content | 内容相同或相似的文件和配置 | 去重处理、存储优化 |
| 事件 | Event | 系统中发生的动作或状态变化 | 事件总线、消息传递 |
| 队列 | Queue | 待处理任务的缓冲区 | 任务调度、异步处理 |
| 工作流 | Workflow | 按特定顺序执行的任务序列 | DAG执行、流程管理 |

## 十二、术语使用检查规则

### 12.1 代码注释检查

```python
# ✅ 正确示例
"""
【类职责】
清理管理器，负责沉积内容清理和重复内容检测
"""

# ❌ 错误示例
"""
【类职责】
清理管理器，负责垃圾文件清理和重复文件检测
"""
```

### 12.2 文档编写检查

```markdown
# ✅ 正确示例
## 仪表盘监控优化
本章节介绍如何优化仪表盘的数据展示和告警机制。

# ❌ 错误示例
## 控制面板监控优化
本章节介绍如何优化控制面板的数据展示和警报机制。
```

### 12.3 变量命名检查

```python
# ✅ 正确示例
alert_threshold = 80  # 【告警阈值】
cleanup_manager = CleanupManager()  # 【清理管理器】

# ❌ 错误示例
warning_threshold = 80  # 应使用"告警"而非"警告"
garbage_collector = CleanupManager()  # 应使用"清理"而非"垃圾"
```

## 十三、术语扩展建议

### 13.1 新增术语流程

1. **提出需求**: 在项目中遇到需要统一的新概念
2. **定义术语**: 明确中文名称、英文对照、定义
3. **审核确认**: 团队讨论确认术语准确性
4. **更新术语表**: 添加到本文档相应章节
5. **同步更新**: 更新相关代码和文档

### 13.2 术语变更流程

1. **评估影响**: 分析术语变更的影响范围
2. **制定计划**: 制定渐进式替换计划
3. **批量替换**: 使用工具批量替换旧术语
4. **验证检查**: 确保替换完整性和正确性
5. **文档更新**: 更新术语表和禁用列表
